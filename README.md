# NexusChat

A production-grade Retrieval-Augmented Generation (RAG) chatbot that answers questions from user-provided documents. Built from scratch using Google Gemini, ChromaDB, hybrid search, cross-encoder re-ranking, and streaming Server-Sent Events — containerised with Docker and tested with GitHub Actions CI.

## Overview

NexusChat ingests PDF, DOCX, and TXT documents, chunks them into searchable segments, and retrieves the most relevant passages at query time using a hybrid search pipeline (dense vector search + BM25 keyword search). Answers are generated by Google Gemini with full source citations, streamed token-by-token to the browser in real time.

## Architecture

| Layer | Technology |
|---|---|
| Document Ingestion | PyPDF2, python-docx, plain text reader |
| Chunking | LangChain RecursiveCharacterTextSplitter (400 chars, 80 overlap) |
| Embeddings | Google Gemini Embedding API (`gemini-embedding-2`) |
| Vector Store | ChromaDB PersistentClient |
| Keyword Search | rank-bm25 BM25Retriever |
| Retrieval Fusion | Reciprocal Rank Fusion (RRF) |
| Re-ranking | sentence-transformers CrossEncoder (`ms-marco-MiniLM-L-6-v2`) |
| Query Rewriting | Google Gemini Flash (standalone query decomposition) |
| Answer Generation | Google Gemini Flash |
| Streaming | Server-Sent Events (SSE) via FastAPI StreamingResponse |
| Chat Persistence | SQLite |
| Backend API | FastAPI + Uvicorn |
| Chat UI | Single-file HTML/CSS/JS with streaming fetch |
| Containerisation | Docker + Docker Compose |
| CI/CD | GitHub Actions |

## Features

- **Hybrid Retrieval** — combines dense vector search and BM25 keyword search via Reciprocal Rank Fusion for higher recall
- **Cross-Encoder Re-ranking** — re-scores retrieved chunks using a MiniLM cross-encoder for precision
- **Query Rewriting** — decomposes follow-up questions into standalone queries using conversation history
- **Streaming Responses** — tokens arrive via SSE as Gemini generates them, with a blinking cursor in the UI
- **Source Citations** — every answer includes clickable source tags with expandable chunk previews
- **Document Upload** — ingest new text documents directly from the browser sidebar
- **Persistent Storage** — chat history in SQLite, vector index in ChromaDB PersistentClient
- **Docker Deployment** — single `docker compose up` starts the full application
- **CI Pipeline** — GitHub Actions runs 10 mocked API tests on every push

## Getting Started

### Prerequisites

- Python 3.11+
- Docker (optional, for containerised deployment)
- A Google Gemini API key

### Local Setup

```bash
git clone https://github.com/Asif3715/Chatbot__internship_AsifKhan.git
cd Chatbot__internship_AsifKhan
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

Create a `.env` file in the project root:

```
GEMINI_API_KEY=your_api_key_here
```

Start the API server:

```bash
python day10/persistent_api.py
```

Open `http://localhost:8000` in your browser.

### Docker Setup

```bash
docker compose build
docker compose up -d
docker compose logs -f
```

Open `http://localhost:8000`. Data persists across container restarts via mounted volumes.

## API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| GET | `/` | Serves the chat UI |
| GET | `/health` | Returns status, chunk count, session count |
| POST | `/chat` | Non-streaming chat (full response) |
| POST | `/chat/stream` | Streaming chat (SSE, token-by-token) |
| POST | `/ingest` | Ingest a new text document |
| GET | `/sessions` | List all chat sessions |
| GET | `/session/{id}` | Get session history |
| DELETE | `/session/{id}` | Delete a session |

### Example: Streaming Chat

```bash
curl --no-buffer -X POST http://localhost:8000/chat/stream \
  -H 'Content-Type: application/json' \
  -d '{"question": "What is NexusChat?"}'
```

## Running Tests

```bash
python -m pytest day12/test_api_ci.py -v
```

All 10 tests run without a Gemini API key (mocked dependencies).

## CI/CD

GitHub Actions workflow (`.github/workflows/ci.yml`) runs on every push:

1. Checks out code
2. Sets up Python 3.11
3. Installs dependencies
4. Runs the mocked API test suite
5. Reports pass/fail with a green tick on the Actions tab
